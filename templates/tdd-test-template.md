# 测试要点清单

> 先列清单，再让 AI 把清单转为测试代码。你不需要会写测试代码，你只需要会列测试要点。

## 模板

对每个功能，按以下格式列出测试要点：

---

### 测试 {序号}: {简短描述}

- **描述**：{一句话说明测什么}
- **前置**：{测试前需要准备什么数据/状态}
- **操作**：{执行什么动作}
- **断言**：{验证什么结果}

---

## 示例：待办事项 — 测试要点清单

### 测试 1: 正常创建待办事项
- **描述**：验证创建待办事项后，标题、优先级、状态都正确
- **前置**：无需特殊准备
- **操作**：创建一个标题为"买牛奶"、优先级为 HIGH 的待办事项
- **断言**：
  - 标题等于"买牛奶"
  - 优先级等于 HIGH
  - 完成状态等于 未完成

### 测试 2: 空标题应报错
- **描述**：验证标题为空时，系统拒绝创建
- **前置**：无需特殊准备
- **操作**：尝试创建标题为空的待办事项
- **断言**：
  - 操作失败
  - 返回错误信息包含"标题不能为空"

### 测试 3: 标记待办事项为已完成
- **描述**：验证可以将待办事项标记为完成
- **前置**：已存在一个未完成的待办事项（标题"测试任务"）
- **操作**：对该事项调用"标记完成"
- **断言**：
  - 该事项的完成状态变为 已完成

---

## 转为测试代码

将上面的清单发给 AI，告诉 AI：
- 编程语言（Python / JavaScript / Java / Go / Rust …）
- 测试框架（pytest / Jest / JUnit / Go testing / cargo test …）
- 要测试的模块/类/函数名

AI 会根据你的清单生成测试代码。

---

## 运行测试

| 语言 | 命令 |
|------|------|
| Python | `pytest tests/ -v` |
| JavaScript | `npm test` |
| Java | `mvn test` |
| Go | `go test ./...` |
| Rust | `cargo test` |
| C# | `dotnet test` |
