# Phase 4: TDD — 先出考题，再写答案

> **把测试当作给 AI 的"考题"——AI 的代码必须通过这些考题才算通过。考题在前，答案在后。**

## 怎么做

1. 为每个 BDD 场景列出测试要点（先不用写代码，写清单即可）
2. 把测试要点清单发给 AI，告诉它你的语言和测试框架，让它生成测试代码
3. 运行测试，确认它们**全部失败**（🔴 Red 状态——因为功能代码还没写）
4. 如果测试直接通过了，说明测试写得有问题，没有真正验证功能

## 测试要点清单格式

```
测试模块：{模块名称}

1. {test_name}
   描述：{一句话说明测什么}
   前置：{测试前需要准备什么}
   操作：{执行什么动作}
   断言：{验证什么结果}

2. {test_name}
   描述：{一句话说明测什么}
   前置：{测试前需要准备什么}
   操作：{执行什么动作}
   断言：{验证什么结果}
```

## 示例：待办事项 — 测试要点清单

```
测试模块：待办事项创建

1. test_create_todo_success
   描述：正常创建待办事项，验证标题和优先级
   前置：无特殊前置条件
   操作：创建标题="买牛奶"、优先级=HIGH 的待办事项
   断言：标题为"买牛奶"，优先级为 HIGH，状态为未完成

2. test_create_todo_empty_title
   描述：空标题创建应被拒绝
   前置：无特殊前置条件
   操作：创建标题=""的待办事项
   断言：操作失败，返回错误"标题不能为空"

3. test_mark_todo_completed
   描述：标记待办事项为已完成
   前置：已存在一个未完成的待办事项
   操作：对该事项调用标记完成方法
   断言：该事项状态变为已完成
```

## 把清单转为测试代码

将上面的清单发给 AI，告诉 AI：
- 你用的编程语言和测试框架（Python/pytest、JS/Jest、Java/JUnit、Go/testing…）
- 要测试的模块/类名

AI 就能根据你的测试要点清单生成测试代码。你不需要会写测试代码，只需要会列测试要点。
